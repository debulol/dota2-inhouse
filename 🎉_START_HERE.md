## 🚀 下一步操作

### 立即开始（15分钟内上线）

#### 第一步：配置 Supabase（5分钟）
1. 访问 https://supabase.com 创建免费账号
2. 创建新项目（选择新加坡区域）
3. SQL Editor 中执行 `supabase/migrations/001_initial_schema.sql`
4. **重要**：修改脚本最后的玩家数据为你的真实数据
5. 复制 Project URL 和 anon key

#### 第二步：本地测试（5分钟）
```bash
cd dota2-inhouse
npm install
cp .env.example .env
# 编辑 .env，填入 Supabase URL 和 Key
npm run dev
```

访问 http://localhost:5173 测试功能！

#### 第三步：部署上线（5分钟）
1. 推送到 GitHub
2. 在 Vercel 导入项目
3. 配置环境变量
4. 点击部署

完成！🎉

## 💡 常见场景

### 场景1：只有我一个人，如何测试？
打开多个浏览器窗口（或不同浏览器），每个窗口选择不同的玩家角色。

### 场景2：如何修改玩家战绩？
在 Supabase 控制台直接修改 `players` 表的数据。

### 场景3：房间满了怎么办？
等待房间自动过期（24小时），或在数据库中手动删除。

### 场景4：想添加更多玩家？
在 Supabase 控制台的 `players` 表中插入新记录。

## 🆘 遇到问题？

### 按顺序检查：

1. **查看 QUICKSTART.md** - 90% 的问题都能找到答案
2. **检查浏览器控制台** - 查看错误信息
3. **查看 Supabase 日志** - 数据库层面的问题
4. **阅读 DEPLOYMENT.md** - 详细的故障排查指南

### 常见问题速查：

- **实时更新不工作** → 检查 Supabase Realtime 是否启用
- **无法加入房间** → 检查玩家是否已在其他房间
- **战绩未更新** → 确认游戏开始时队伍都是5人
- **页面空白** → 检查环境变量是否配置正确
